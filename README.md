# For You, Rhia — An Interactive 3D Love Letter

A bespoke, single-page interactive 3D experience built as a personal love letter. The site combines a WebGL scene (React Three Fiber + Three.js), GSAP/scroll-driven storytelling, and a handful of themed "portal" sections (timeline, gallery, reasons, dreams, letter).

## Tech Stack

- **Framework**: TanStack Start v1 (React 19, file-based routing in `src/routes/`)
- **Build**: Vite 7, deployed to Cloudflare Workers (`wrangler.jsonc`)
- **3D**: Three.js, `@react-three/fiber`, `@react-three/drei`, `@react-three/postprocessing`
- **Animation**: GSAP + `@gsap/react`, ScrollTrigger
- **Styling**: Tailwind CSS v4 (tokens defined in `src/styles.css`, oklch color space)
- **UI primitives**: shadcn/ui (Radix) components in `src/components/ui/`
- **State**: Zustand stores in `src/stores/`
- **Backend (optional)**: Lovable Cloud (Supabase) — server functions under `src/lib/*.functions.ts`

## Notable Features

- **`PortfolioScene`** orchestrates the full WebGL canvas, scroll wrapper, and accessible portal navigation (sr-only buttons + `<noscript>` fallback links).
- **`CanvasLoader`** applies a dreamy postprocessing stack: ACES filmic tone mapping, heavy Bloom, Depth of Field, soft Vignette.
- **`Encounter` model** uses a module-level URL-keyed cache (`urlCache: Map<url, { scene, materials, textures, geometries, bySource }>`) to memoize cloned GLTF scenes and remapped `MeshBasicMaterial` instances across re-renders. Includes a `disposeEncounterCache(url)` cleanup that disposes materials/textures/geometries when leaving the gallery or switching models. Handles both `Mesh` and `SkinnedMesh` (via `SkeletonUtils.clone`) and nested material arrays.
- **Gallery** features a dense white exponential fog atmosphere, third-person camera with `lookAt`-based level framing, contact shadows, and a lightbox.
- **Portal store** drives transitions between the in-canvas world and HTML overlays (timeline, gallery, reasons, dreams, letter).

## Deployment

Configured for Cloudflare Workers via `@cloudflare/vite-plugin` and `wrangler.jsonc`. SSR runs on workerd with `nodejs_compat`.

## Notes

- Do not edit `src/routeTree.gen.ts` — it is regenerated by the TanStack Router Vite plugin.
- All design tokens live in `src/styles.css`; never hardcode colors in components.
- The `node_modules/`, `.lovable/`, build outputs, and lockfiles are excluded from the distributed zip.
